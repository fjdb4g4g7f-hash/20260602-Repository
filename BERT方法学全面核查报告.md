## 二、🔴 致命问题（2个）

### 问题1：统计显著性检验完全缺失

**代码证据**：在所有15个Python文件中搜索 `ttest`、`p_value`、`scipy.stats`、`statistical`、`significance`、`wilcoxon` — **零匹配**。

**论文声称**：方法部分写明"采用配对样本t检验，验证事件前后各项指标差异的统计显著性"

**实际情况**：动态语义分析（Step 3）仅计算了事件前后的余弦距离描述性数值，画了柱状图，但从未计算p值、置信区间或任何推断统计量。

**为什么致命**：
- 论文的核心实证发现——"关键事件显著改变了中菲叙事语义距离"——完全建立在描述性统计之上
- 某些"变化"仅为0.003-0.004的余弦距离差异，在小样本窗口下极可能是随机噪声
- 所有关于"焦点收敛效应""叙事错位加剧/收窄"的因果性表述缺乏统计支撑

**修复建议**：
1. 在Step 3中实现配对样本检验（t检验或Wilcoxon符号秩检验）
2. 报告Bootstrap置信区间
3. 计算效应量（Cohen's d）
4. 对5个事件的多重比较进行Bonferroni校正
5. 论文中报告精确p值和置信区间

## 三、🟠 严重问题（7个）

### 问题3：稳定性检验逻辑存在根本错误

**代码证据**（`脚本C-完整输出.py` 第92-107行）：
```python
# 稳定性检验的模型只做了 auto 合并
tm_s.reduce_topics(docs, nr_topics="auto")
n_s = len([t for t in set(tm_s.topics_) if t != -1])
```

**但主模型（seed=42）的流程是**：
```python
# 脚本B：auto → 若>20 → 强制合并至15
topic_model.reduce_topics(docs, nr_topics="auto")
if AUTO_COUNT > 20:
    topic_model.reduce_topics(docs, nr_topics=15)
```

**为什么严重**：
- 稳定性测试和主模型走了不同的合并逻辑
- 如果seed=0的auto结果是22个主题，seed=100是18个，主模型被强制到15个
- 比较三个不同合并深度下的主题数量毫无意义
- 论文声称"结果稳定"的结论不成立

**修复建议**：
1. 稳定性测试的每个seed必须复制主模型的完整pipeline（auto → 判断 → 强制合并）
2. 计算主题分配一致性（NMI/ARI），而非仅比较数量
3. 使用5-10个不同seed

### 问题4：数据量存在严重不一致

**代码证据**：
- `脚本A` 第3行注释："把5800条文本编码成向量"

**矛盾分析**：

**为什么严重**：
- 样本流失可能不是随机的（如某国/某时期的文章被系统性排除）
- 40%的数据损耗如果不加说明，会严重影响研究可复现性

**修复建议**：
1. 文档化完整数据流失链条：收集(11325) → 去重 → 缺失值过滤 → 短文本过滤 → 最终入模数
2. 报告每一步的样本量和过滤标准
3. 如存在版本差异（Excel文件更新），予以说明

### 问题5：主题合并过于随意

**代码证据**（`脚本B` 第96-101行）：
```python
if AUTO_COUNT > 20:
    FINAL_TARGET = 15   # 硬编码的阈值
    topic_model.reduce_topics(docs, nr_topics=15)
```

**论文声称**：
> "首先应用自动层次主题降维将结果收敛至51个主题；随后合并为15个"

**问题**：
- 从51个主题压缩到15个，减少了70%，这是极大的跳跃
- 为什么是15？不是12、18或20？没有任何系统性标准
- `reduce_topics(nr_topics=15)` 使用c-TF-IDF向量相似度合并，在此极端压缩下可能将语义不同的主题强行合并
- 网格搜索仅覆盖9种参数组合（n_neighbors × min_cluster_size = 3×3），范围很小

**为什么严重**：
- 论文的4大类和12个有效主题的分类体系，高度依赖于15这个数字
- 如果合并到12个主题或18个主题，分类结果可能显著不同
- 后续所有基于主题的分析（分布对比、时间序列）都受此影响

**修复建议**：
1. 扩展网格搜索参数范围（n_neighbors: 5-30, min_cluster_size: 10-50）
2. 以主题间语义相似度阈值代替固定目标数量
3. 报告层次合并树（dendrogram）
4. 进行敏感性分析：用12、15、18个主题分别跑核心分析，验证结论稳健性

### 问题6：稳定性检验仅比较主题数量

**代码证据**（`脚本C` 第105行）：
```python
n_s = len([t for t in set(tm_s.topics_) if t != -1])
# 仅比较了主题数量！
```

**实际学术标准**：
- 两个模型都产出15个主题，不意味着主题内容相同
- 需要比较文档-主题分配的一致性：NMI（归一化互信息）、ARI（调整兰德指数）
- 需要比较主题词语义重叠度（Jaccard相似度）

**为什么严重**：
- 论文的"结果稳定"声明缺乏实质性证据
- 主题模型的不稳定性是已知问题，仅靠数量比较无法排除

**修复建议**：
1. 计算5-10个seed之间的NMI和ARI
2. 报告主题关键词的Jaccard重叠度
3. 论文中诚实报告稳定性指标的实际数值

### 问题8：叙事重心锚词选择存在系统性偏差

**代码证据**（`Step 3` 第181-190行）：
```python
CONFRONTATION_ANCHORS = [
    "military confrontation aggression threat provocation illegal violation",
    "coast guard vessel water cannon harassment dangerous",
    "sovereignty violation armed attack escalation",
]
COOPERATION_ANCHORS = [
    "peaceful dialogue cooperation stability regional development",
    "ASEAN partnership bilateral consultation agreement",
    "negotiation diplomatic channel de-escalation",
]
```

**问题**：
- 对抗锚词：具体、多样、富含动作词汇（military, water cannon, armed attack）
- 合作锚词：抽象、概括（stability, development, partnership）
- 在语义向量空间中，抽象词汇天然更分散，可能导致"合作"端距离更大
- 锚词是人工主观定义的，未经任何语料库验证

**为什么严重**：
- 叙事重心得分（Score = Sim_confront - Sim_coop）的所有值都依赖这些锚词
- 如果换一套锚词，结果可能完全不同
- 图6的"叙事重心漂移"箭头方向可能反映锚词选择偏差而非真实叙事变化

**修复建议**：
1. 用c-TF-IDF从实际语料的对抗性/合作性主题中提取锚词
2. 进行敏感性分析：用3-5套不同锚词计算，报告分数范围
3. 考虑使用已验证的词典（如LIWC、ConfliBERT）
4. 论文中报告完整锚词列表

## 五、🟢 轻微问题（5个）

### 问题16：词边界匹配遗漏形态变体

**代码**（`语义空间层.py` 第66行）：
```python
re.search(r'\b' + re.escape(word) + r'\b', sent_lower)
```

`\b` 只精确匹配，如 "defense" 不会匹配 "defensive"、"self-defense"。

**修复**：使用词形还原（lemmatization）后进行匹配，或扩展正则覆盖常见变体。
